# Market Brief 설계 문서

뉴스 기반 주가 영향(호재/악재) 분석 백엔드

## LLM 연동 방식 옵션

| 방식 | 장점 | 단점 |
|------|------|------|
| **OpenAI API** | 안정적, 문서 풍부 | 유료, 레이턴시 |
| **Anthropic API** | 긴 컨텍스트, 분석 강점 | 유료 |
| **Ollama (로컬)** | 무료, 프라이버시 | 성능 제한, 서버 리소스 필요 |
| **Google Gemini** | 무료 티어 있음 | API 제한 |

## 구현 우선순위

```
1. 뉴스 수집 (입력 없으면 분석 불가)
   └─ 크롤링 or 뉴스 API 연동

2. LLM 연동 (핵심 기능)
   └─ 호재/악재 판단 프롬프트 설계

3. DB 연동 (결과 저장)
   └─ PostgreSQL or MongoDB

4. 종목 관리 (부가 기능)
   └─ 티커 매핑, 관심 종목 등
```
